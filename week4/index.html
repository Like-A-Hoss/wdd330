Object Oriented Familarity

After getting my Programming certification I laugh seeing this. Because here is something I know quite well my old friend Object Oriented Programing. A good old friend from my days of writting apps in Java and Python programs. So let's do a quick run down of some Vocabularity!

  • Object Oriented Programing - A programing method that uses "objects" that model reworld objects, including living beings. A cat can be modled as an object, in a program for example.
  • Encapsulation - A property of object oriented programing. All objects contain or encapsulate and protect their contents from interfearance by things outside them. Similar to how skin protects the innards of living critters, and keeps everything inside.
  • Polymorphism - Aside from proof that at least the early object oriented programmers were nerds, Polymorphism is the concept of taking something and turning it into something else similar to the Dungeons and Dragons spell by that name, which takes one living critter and turns it into another, though another comparison comes from adaptation in living creatures. Polymorphism can also be compaired to how pigeons in New York can differe from Pigeons in Tokyo. Both are still pigeons, but they have been apated to new tasks.
  • Inheritance - Taking someone and adding new features to it to make it soemthing new. In comparison to that cat, it the process that takes a lizard, and though a series of inheritances keeping features and adding and polymorphing others we eventually get a cat.
