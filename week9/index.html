DOM, Window Object, and Global Constants/Varibles

The Window

The window is a global object that exists solely in the context of browers. Thus for other applications like a discord chatbot, toasters, micowaves, and anything that is cellphone apps, or doing anything that is not a application to be run in a web browser do not act as if this exists.

Global varibles for we development are placed here. global varibles are declared without using const, let, var ect. Just something like x=7 they can be retrieved like this window.x

parseInt() and isNaN() are also a part of the window object. isNaN is likely good for error trapping. Best to keep that one in mind.

Dialogs

dialog boxes are also a part of the window object so alert(), confirm() and prompt() are all children of the window.

It might be well worth noting that some of these stop everything else from running like confirm().

Browser Info

Should you need it you can even get the informaiton aobut the client's browser. This can be useful for allowing you the developer to ensure backwords compatiblility with older browsers.
